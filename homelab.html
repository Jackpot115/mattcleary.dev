Personal Homelab

8.16.25

Shout out to:

  • Ardens for the inspiration on what to self-host.
  • HardwareHaven for the helpful guides and tutorials on how to find and salvage computers and hardware.
  • Louis Rossmann for his incredible FUTO guide.

If it weren't for these creators, I wouldn't have been able to build my homelab...

I took about a week doing research and finding out what I could do to upgrade my already growing homelab. I discovered these Youtube channels and found inspiration through all of them.

I first went out and bought myself more ethernet cables for my network switch. This was just so I can have more ports for my devices on my homelab's network. Additionally, I remembered that I had a Raspberry Pi 5 laying around and learned from Louis Rossmann's FUTO guide that I could set it up to run Pi-hole, a network-wide ad blocker.

The Lenovo Mini PC continued to be just a regular Windows 11 PC running a Minecraft Paper server. I want to get mods running on the server, but need to learn how to get them to work on a small device like the Mini PC.

I took inspiration from HardwareHaven and bought myself a HP Mini PC on ebay. Upgrading the ram inside of it to 8GB, I made the PC a Ubuntu server and installed Docker Compose to run some Docker containers. Some containers I set up for starters were:

  • Home Assistant
  • Portainer
  • Uptime Kuma

Lastly, I found a stand for my Lenovo Mini PC to keep it upright. Everything fit perfectly on top of the TrueNAS server I had set up. Things are going great!

8.7.25

I taught myself how to image a new operating system onto a computer to repurpose it for my homelab. I discovered about the Network Attached Storage (NAS) and how it can be used to store and share files across a network. Now, I could take the 500GB hard drive that was plugged into the Lenovo Mini PC and teach myself how to set it up through TrueNAS.

I successfully was able to hook the TrueNAS up to the salvaged laptop I had and could connect to the GUI from my network. Here, I taught myself how to create a storage disk, datasets, pools, shares, and most importantly, data protection. Using TrueNAS made sharing the storage device a lot easier since it was through the GUI. Not only could I share the disk via SMB, but I could also share it via NFS!

However, I ran into a problem. I couldn't close the laptop lid or else the computer would shut down or the text from the screen would burn into the display. So, I taught myself how to pop open the hood on the laptop and disconnect the screen power from the motherboard.

Popping open the hood helped me understand how the inside of a laptop was laid out. Not only did I learn that the laptop had a M.2 drive for extra storage, but I also learned that with laptops, the CPU is usually soldered into the motherboard! Unhooking the connection to the display screen not only saved the screen from burning but also helped conserve energy.

This allowed my first complete server to stay on while the screen was turned off. Now, if I needed to do anything with the server, I could access it via my personal laptop either through SSH or through the TrueNAS GUI!

8.6.25

The start of my homelab journey.

I decided to start a homelab after I was gifted a Lenovo Mini PC. At first, I just wanted to run a Minecraft server for my college friends, but this quickly introduced me to the world of self-hosting.

In the image: a wired keyboard and mouse, the Lenovo Mini PC, a small layer 2 network switch, a Raspberry Pi 5, and a 500GB hard drive salvaged from another laptop. I taught myself how to upgrade the Lenovo from Windows 10 to Windows 11, bypassing the CPU requirements via command prompt. I also set up the hard drive as a shared network drive using SMB and got Raspberry Pi OS running on the Pi.

I quickly got hooked on self-hosting and started exploring different services and hardware options at various price points.
